## 2. Локализация данных (ст. 18 ч. 5 152-ФЗ)

Все персональные данные граждан РФ хранятся **исключительно** на территории Российской Федерации:

| Компонент | Где физически | Тип данных |
|---|---|---|
| PostgreSQL 16 (основная БД) | Сервер в МО, РФ (Россия) | Учётные записи, события L1-L3, метаданные |
| MinIO (object storage) | Сервер в МО, РФ | Медиа-файлы, снапшоты L4 |
| Redis (кеш + сессии) | Сервер в МО, РФ | Активные сессии, KNN-кеш |
| Резервные копии (rclone offsite) | Опциональный второй сервер в РФ | Шифрованные снапшоты |
| Email (Yandex 360) | mail.yandex.ru (РФ) | Логин-токены OTP, transactional emails |

**Транзит за пределы РФ:** не осуществляется. Внешние LLM-провайдеры (DeepSeek, OpenAI, Claude) подключаются **по выбору tenant'а** через его собственные API-ключи (`/ui/profile` → External Keys); по умолчанию выключены. При использовании внешнего LLM субъект данных должен быть отдельно уведомлён tenant'ом — это его ответственность как контролёра данных. Платформа поддерживает «чисто-РФ» режим: GigaChat (Сбер) + YandexGPT — оба data-residency РФ.

| Цель | Категория ПД | Основание | Срок хранения |
|---|---|---|---|

| Доставка email-уведомлений | Email | Договор-оферта (cl. 4) | До отзыва согласия |

| Биллинг (платные тарифы) | Email, ИНН, реквизиты счёта | Закон 54-ФЗ (фискализация) | 5 лет (НК РФ) |
| Безопасность (логи доступа) | IP, user-agent, время | Законный интерес оператора | 12 месяцев |
| Резервные копии | Все категории выше | Законный интерес оператора | 30 дней (hot) + 90 дней (cold) |

## 4. Категории субъектов

Платформа предназначена для **B2B** (юридические лица + ИП). Контролёр (controller) — клиент-компания, использующая платформу для своих агентов. Платформа выступает в роли **процессора** (processor) согласно ст. 6 152-ФЗ.

Клиент-tenant обязан:
1. Получить отдельное согласие у конечных физических лиц, чьи данные попадают в память агента
2. Не размещать в платформе спецкатегории ПД (медицинские, биометрические, политические, религиозные) без отдельного согласования
3. Подписать с платформой **Договор о поручении обработки ПД** (шаблон: [docs/dpa-template-152fz.md](dpa-template-152fz.md) — будет доступен с 2026-06-01)

## 5. Технические меры защиты (ст. 19 + Приказ ФСТЭК №21)

| Мера | Реализация | Уровень защищённости |
|---|---|---|
| **Шифрование канала** | TLS 1.3 (Let's Encrypt, HSTS) | УЗ-3 |
| **Шифрование at-rest (опционально)** | LUKS на диске сервера (по запросу enterprise tenants) | УЗ-2 |
| **Шифрование внешних API-ключей** | Fernet (AES-128 + HMAC-SHA256) в таблице `user_external_keys` | УЗ-2 |
| **Аутентификация** | Email OTP (15-мин TTL, 6-значный, single-use, SHA-256 хеш в БД, rate-limit 5/час) | УЗ-3 |
| **Авторизация** | Per-agent X-API-Key (32-байтовый токен), bcrypt-хеш в БД | УЗ-3 |
| **Изоляция tenants** | `owner_user_id` UUID в WHERE на всех запросах L1/L2/L3, per-owner prefix в MinIO | УЗ-3 |
| **Аудит** | Все события access/auth/admin → таблица `l5_audit_events`, retention 12 мес | УЗ-3 |
| **Rate-limiting** | Redis-based 100 req/s per agent + per-tier quotas | УЗ-3 |
| **Защита от SQL injection** | asyncpg parameterized queries ($1, $2) — никогда string-concat | УЗ-3 |
| **Защита от XSS/JSON injection** | Pydantic strict validation + `sanitize_payload` middleware | УЗ-3 |
| **Backup** | Daily snapshot, hot 30d (NVMe) + cold 90d (HDD), шифрование gpg | УЗ-3 |
| **Резервный канал доступа** | OOB SSH 2222 (изолированный от primary, fail2ban) | УЗ-3 |
| **Мониторинг** | Prometheus + Grafana + Loki + alerting (auth_failure, rate spikes) | УЗ-3 |
| **Контейнерная изоляция** | Docker, нет privileged containers, нет host-network | УЗ-3 |
| **Firewall** | UFW: только 22/80/443 публично, всё остальное в bridge | УЗ-3 |
| **Управление уязвимостями** | Trivy weekly scan + unattended-upgrades для Ubuntu | УЗ-3 |
| **Удаление по запросу** | SQL DELETE + восстановление невозможно после 30 дней | УЗ-3 |

**Уровень защищённости:** УЗ-3 для типовой ИСПДн (без спецкатегорий ПД и без биометрии). При необходимости УЗ-2 — on-premise deployment с дополнительными мерами заказчика.

## 6. Права субъекта ПД (ст. 14-18)

Субъект может реализовать следующие права через email **privacy@me-ai.ru** или личный кабинет `/ui/profile`:

| Право | Способ реализации | SLA |
|---|---|---|
| Доступ к своим ПД | GET `/user/me` + GET `/user/data-export` (JSON-выгрузка всех связанных записей) | До 30 дней (ст. 14) |
| Уточнение/изменение | PATCH `/user/me` + UI `/ui/profile` | Мгновенно |
| Блокировка обработки | POST `/user/suspend` (отключает все агенты, оставляет данные) | До 5 рабочих дней |
| Удаление (право на забвение) | DELETE `/user/account` → каскад на все таблицы + MinIO + восстановление невозможно через 30 дней | До 30 дней (ст. 21) |
| Отзыв согласия на обработку | Тот же DELETE `/user/account` | До 30 дней |
| Жалоба в Роскомнадзор | Контакты на rkn.gov.ru | — |

При обнаружении факта несанкционированного доступа или утечки ПД — оператор обязан в течение **24 часов** с момента обнаружения уведомить:

1. Субъектов ПД (всех затронутых) — через email на их зарегистрированный адрес
2. Роскомнадзор — через rkn.gov.ru/notification

Контактное лицо: dpo@me-ai.ru.

## 8. Чек-лист соответствия 152-ФЗ для tenant'ов

Если вы используете Cognitive Core для обработки ПД ваших клиентов:

- [ ] Подписан **Договор о поручении обработки ПД** с оператором платформы
- [ ] Получено согласие от ваших конечных пользователей на обработку их ПД (вы — контролёр, мы — процессор)
- [ ] В вашей Политике конфиденциальности указано использование Cognitive Core как процессора
- [ ] Не записываются спецкатегории ПД в память агента без отдельного согласования
- [ ] Включён 2FA на вашем кабинете оператора (для платных тарифов — обязателен с 2026-07)
- [ ] Настроены ротация API-ключей (рекомендация — каждые 90 дней)
- [ ] Резервные копии в собственной инфраструктуре (опционально, на платных тарифах — autobackup на R2/S3 owner)
