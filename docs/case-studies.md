# Cognitive Core — примеры применения (illustrative scenarios)

> **⚠️ Важно — это illustrative scenarios, а не реальные клиенты.**
> На момент публикации (2026-05) у Cognitive Core нет публичных клиентов, чьи
> кейсы мы могли бы цитировать. Всё ниже — **синтетические, но реалистичные
> сценарии использования**, построенные на реальных возможностях платформы.
> Метрики помечены как «illustrative» — это ожидаемые/смоделированные значения,
> **не измеренные у реальных пользователей**. Мы намеренно не выдумываем
> отзывы и не приписываем цитаты несуществующим людям.

Эти сценарии показывают, как складываются ключевые фичи платформы — 5-слойная
память, multi-agent rooms, РФ-compliance, media pipeline — в типовых рабочих
ситуациях. Используйте их как карту: «похоже на мой кейс → вот какие
инструменты применять».

---

## Сценарий 1 — Команда разработки на нескольких агентах

*Illustrative scenario.*

### Контекст

Соло-разработчик (или маленькая команда) держит несколько AI-ассистентов:

- Claude Code на рабочем ноутбуке
- Claude Code на домашнем десктопе
- Cursor для фронтенда
- ChatGPT Custom GPT для ресёрча и черновиков

Все они — агенты **одного owner-а**, работают над одним продуктом.

### Проблема (как было без Cognitive Core)

Каждый агент стартует «с чистого листа». Решение, найденное вчера в Claude Code
на ноутбуке (например, обходной путь для бага в деплое), **недоступно** Cursor-у
сегодня. Знание живёт в локальной памяти каждого инструмента и не шарится.
Результат — повторное решение тех же проблем, рассинхрон контекста, копипаста
заметок между чатами вручную.

### Решение (какие фичи)

1. Все четыре агента подключаются к одному owner-аккаунту через `/ui/connect`
   (claim-token флоу).
2. После значимой задачи агент пишет вывод в память:
   ```python
   cognitive_remember(
       domain="deploy",
       task="Auto-deploy падал на rebuild api (IPv6 к registry)",
       result="Workaround: docker pull python:3.11-slim заранее, "
              "затем compose build берёт из кеша",
       lessons="Permanent fix — daemon.json ipv6 false",
       tools_used="docker",
   )
   ```
3. Перед новой задачей **любой** агент сначала ищет прошлый опыт:
   ```python
   cognitive_recall(query="ошибка деплоя api rebuild", top_k=5)
   ```
4. Сырые события (L1) консолидируются ночью в L3-знание (DeepSeek-куратор
   отсеивает шум — болтовню, неудачные попытки, дубли), так что `recall`
   возвращает дистиллят, а не сырой лог.
5. Для живой координации между агентами — shared room:
   ```python
   room_join(room_key="dev-team")
   room_post(room_key="dev-team", text="Залил фикс деплоя, см. domain=deploy")
   ```

### Результат (illustrative)

| Метрика | До | После | Примечание |
|---|---|---|---|
| Повторно решённые задачи / неделя | ~5 | ~1 | illustrative |
| Время на «вспомнить контекст» в начале задачи | 10-15 мин | <2 мин | illustrative |
| Источник истины по решениям | разрозненные локальные заметки | единый L3 per-owner | — |
| Онбординг нового агента в проект | с нуля | мгновенный recall накопленного | — |

**Суть**: `cognitive_recall` перед задачей превращает четыре изолированных
ассистента в одну команду с общей памятью. Подробнее про scope — какой агент
что видит — в `docs/memory-scope.md`.

## Сценарий 2 — РФ enterprise под 152-ФЗ

*Illustrative scenario.*

### Контекст

Российская компания (финтех / госсектор / медицина) хочет дать своим внутренним
AI-агентам долговременную память, но связана требованиями 152-ФЗ: персональные
данные **не должны покидать территорию РФ**.

### Проблема (как было без Cognitive Core)

ChatGPT Memory и облачные memory-сервисы (Mem0 Cloud, Zep Cloud) хостятся за
рубежом. Отправка туда ПДн = нарушение 152-ФЗ. Без legal-«бумажки» enterprise
не может даже начать пилот. В итоге — либо никакой персистентной памяти у
агентов, либо «теневое» использование зарубежных сервисов с юридическим риском.

### Решение (какие фичи)

1. **Self-hosted инстанс** Cognitive Core на инфраструктуре компании (или в
   РФ-облаке: Selectel / Yandex Cloud / Timeweb) — один docker-разворот:
   ```bash
   curl -fsSL https://mcp.me-ai.ru/static/install-self-hosted.sh | sudo bash
   ```
   Postgres, Redis, MinIO, API — всё внутри периметра. Данные не уходят наружу
   (см. `docs/onboarding-vps.md`).
2. **РФ-резидентные AI-провайдеры**: GigaChat (Сбер) и YandexGPT адаптеры для
   vision/LLM-задач — иностранные модели не нужны.
3. **Compliance-pack**: `docs/compliance-152fz.md` описывает data residency,
   технические меры по Приказу ФСТЭК-21 (УЗ-3, при on-premise — УЗ-2), права
   субъектов ПДн (доступ/удаление/блокировка) с конкретными endpoints. Плюс
   `docs/dpa-template-152fz.md` — черновик договора поручения обработки ПДн.
4. **Tenant isolation**: per-owner `owner_user_id` WHERE-фильтры на L1/L2/L3 +
   per-owner MinIO prefix — данные разных подразделений не пересекаются.

### Результат (illustrative)

| Аспект | До | После |
|---|---|---|
| Юридический статус памяти агентов | блокер (ПДн за рубеж) | соответствует 152-ФЗ |
| Где живут данные | — (нельзя использовать) | в периметре РФ, под контролем компании |
| LLM-провайдер | OpenAI/Anthropic (запрещены для ПДн) | GigaChat / YandexGPT |
| Время до старта пилота | бессрочно заблокировано | self-host разворот + DPA на ревью юриста |

**Суть**: self-host + РФ-провайдеры + готовый compliance-pack снимают
юридический блокер, который делает большинство зарубежных memory-сервисов
неприменимыми в РФ-enterprise.

> **Дисклеймер по compliance**: документы (`compliance-152fz.md`,
> `dpa-template-152fz.md`) — это шаблоны и технические описания, **не замена
> юридической экспертизы**. Финальную редакцию DPA и оценку соответствия должен
> делать ваш юрист / DPO.

## Сценарий 3 — Соло-исследователь с media-heavy потоком

*Illustrative scenario.*

### Контекст

Независимый исследователь (или аналитик / журналист) генерирует много
**аудио- и видео-материала**: записи интервью, голосовые заметки, скринкасты
с лекций, видео-дневник наблюдений. Накапливаются месяцы материала.

### Проблема (как было без Cognitive Core)

Видео и аудио — «тёмные данные»: их нельзя искать текстом. Чтобы найти, где три
месяца назад собеседник упомянул конкретную цифру, приходится вручную
переслушивать часы записей. Knowledge заперт в непоисковом формате.

### Решение (какие фичи)

1. Media pipeline через CLI `cogmedia`:
   ```bash
   COGNITIVE_API_KEY="$KEY" cogmedia upload "/path/interview_2026-03-12.mp4"
   ```
   Сервер: видео → ffmpeg извлекает ключевые кадры (1 fps) + **Whisper-транскрипт**;
   аудио → Whisper-транскрипт. Метаданные навсегда оседают в L1
   (`domain=media_analysis`), кадры доступны через media-frame endpoint.
2. Транскрипты проходят ту же консолидацию L1→L3, что и обычные события —
   становятся **частью единой поисковой базы знаний**.
3. Поиск по месяцам материала — одним вызовом:
   ```python
   cognitive_recall(domain="media_analysis",
                    query="собеседник про долю рынка в Q1")
   ```
4. Для агента, читающего видео: флоу `cogmedia upload` → frames → `Read`
   позволяет ассистенту буквально «посмотреть» ключевые кадры, а не только текст.

### Результат (illustrative)

| Метрика | До | После | Примечание |
|---|---|---|---|
| Поиск факта в архиве записей | часы переслушивания | секунды (`recall`) | illustrative |
| Формат знания | непоисковое видео/аудио | searchable L3-транскрипты | — |
| Переиспользование старых интервью | редко (дорого искать) | регулярно | illustrative |
| Связь медиа с текстовыми заметками | разорвана | единый knowledge-слой | — |

**Суть**: media pipeline превращает «тёмные» часы записей в queryable знание.
Месяцы исследований становятся доступны одним `cognitive_recall`.

## Как читать эти сценарии

- Это **карта применения фич**, не доказательство. Метрики — illustrative
  (ожидаемые/смоделированные), не измеренные у реальных клиентов.
- Реальный эффект зависит от вашего объёма данных, дисциплины записи
  (`remember` после задач) и того, насколько ваш кейс попадает в нишу
  «MCP + multi-agent + консолидация + РФ + self-host».
- Где Cognitive Core **не** оптимален — честно описано в
  `docs/migration-from-competitors.md`, раздел «Когда НЕ выбирать».

Когда появятся реальные публичные кейсы с измеренными метриками и согласием
клиентов на публикацию — заменим эти illustrative scenarios на настоящие.
